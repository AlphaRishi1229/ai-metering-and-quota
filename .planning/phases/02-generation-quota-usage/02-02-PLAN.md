<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → GenerateRequest | Untrusted prompt string crosses into the application |
| app → Postgres | UsageLog rows written with user_id from path parameter |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Tampering | GenerateRequest.prompt (unbounded string) | mitigate | min_length=1 rejects empty; no max_length imposed — Plan 03 can add if needed; provider estimate bounds cost before generation |
| T-02-02-02 | Information Disclosure | UsageLog stores prompt_tokens + completion_tokens (not prompt text) | accept | D-20 explicitly prohibits storing prompt text; no PII in log row |
| T-02-02-03 | Tampering | user_id FK from path parameter | mitigate | Plan 03 does DB lookup to confirm user exists before writing UsageLog; FK constraint at DB level prevents orphan rows |
| T-02-02-04 | Elevation of Privilege | status field accepts arbitrary string | mitigate | Plan 03 router only writes literal "success", "quota_exceeded", "ai_error" — no user-controlled input flows into status |
</threat_model>
