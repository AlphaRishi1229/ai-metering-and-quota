  ```python
  from app.config import settings
  ```

  **app/models.py** — User ORM model with all Phase 1 + Phase 2 columns (per D-11, D-12, CONTEXT specifics):
  ```python
  from sqlalchemy import Integer, Float, BigInteger
  from sqlalchemy.orm import Mapped, mapped_column

  from app.database import Base


  class User(Base):
      __tablename__ = "users"

      id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
      quota: Mapped[int] = mapped_column(Integer, nullable=False)
      multiplier: Mapped[float] = mapped_column(Float, nullable=False)
      used_credits: Mapped[int] = mapped_column(BigInteger, nullable=False, default=0)
      reserved_credits: Mapped[int] = mapped_column(BigInteger, nullable=False, default=0)
  ```

  Column rationale:
  - `id`: auto-increment integer (simpler than UUID for an interview project, clear in URLs)
  - `quota`: Integer (credits are whole numbers, per D-11 gt=0)
  - `multiplier`: Float (per D-12; token estimates are approximate so NUMERIC precision is false accuracy)
  - `used_credits`, `reserved_credits`: BigInteger (can grow large over many requests; default 0, no server_default needed since create_all inserts the Python default)

  **app/schemas.py** — Pydantic schemas (per D-05, D-06, D-07, D-11, D-12, CONTEXT response fields):
  ```python
  from pydantic import BaseModel, Field


  class UserCreate(BaseModel):
      quota: int = Field(gt=0)
      multiplier: float = Field(gt=0.0)


  class UserUpdate(BaseModel):
      quota: int | None = Field(default=None, gt=0)
      multiplier: float | None = Field(default=None, gt=0.0)


  class UserResponse(BaseModel):
      id: int
      quota: int
      multiplier: float
      used_credits: int
      reserved_credits: int

      model_config = {"from_attributes": True}
  ```

  Note: `UserUpdate` has both fields optional so either or both may be provided (per D-05). If both are None, PATCH is a no-op (returns current user unchanged — acceptable, no need for a special 400).

  **app/main.py** — FastAPI app with lifespan-based table creation (per D-01, D-08):
  ```python
  from contextlib import asynccontextmanager

  from fastapi import FastAPI

  from app.database import engine, Base


  @asynccontextmanager
  async def lifespan(app: FastAPI):
      async with engine.begin() as conn:
          await conn.run_sync(Base.metadata.create_all)
      yield


  app = FastAPI(title="AI Metering Service", lifespan=lifespan)
  ```

  Note: `conn.run_sync(Base.metadata.create_all)` is the correct async pattern — `create_all(bind=engine)` is the sync API. The async engine requires `run_sync` inside `engine.begin()`.

  **docker-compose.yml** — Postgres with healthcheck + app depends_on service_healthy (per D-09):

  ```yaml
  version: "3.9"

  services:
    db:
      image: postgres:16-alpine
      environment:
        POSTGRES_USER: postgres
        POSTGRES_PASSWORD: postgres
        POSTGRES_DB: metering
      healthcheck:

      ports:
        - "8000:8000"
      depends_on:

          condition: service_healthy
      command: uvicorn app.main:app --host 0.0.0.0 --port 8000
  ```
